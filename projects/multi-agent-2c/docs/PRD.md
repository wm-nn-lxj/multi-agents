## 1. 产品概述

### 1.1 产品定位
构建一个面向2C软件产品的多Agent协作框架，通过5个专业化Agent协同工作，实现从需求分析到测试验收的完整软件交付流程。

### 1.2 核心价值
- **自动化交付**：减少人工干预，提升软件交付效率
- **专业化分工**：每个Agent专注特定领域，提升输出质量
- **流程可控**：明确的检查点和人工审核机制
- **上下文优化**：控制在16K以内，保证响应速度和成本控制

### 1.3 目标用户
- 软件开发团队
- 产品经理
- 技术创业者
- 需要快速原型验证的团队

## 3. 技术架构

### 3.1 框架选型

基于业界调研，选择 **LangGraph** 作为核心框架：

| 框架 | 优势 | 劣势 | 适用场景 |
|:---|:---|:---|:---|
| **LangGraph** ✅ | 生产就绪、状态管理强、流程可控 | 学习曲线陡峭 | 复杂工作流、生产环境 |
| AutoGen | 简单易用、快速原型 | 稳定性不足 | 快速验证、实验项目 |
| CrewAI | 角色化协作、专业分工 | 生态较小 | 多角色协作场景 |

**选择理由**：
1. 生产环境首选，稳定性最高
2. 基于图结构，流程可控性强
3. 状态管理完善，支持上下文压缩
4. 社区活跃，文档完善

### 3.3 通信机制

| 通信类型 | 实现方式 | 用途 |
|:---|:---|:---|
| Agent间通信 | LangGraph State | 任务传递、结果共享 |
| 上下文传递 | 压缩后的State | 保持上下文连续性 |
| 记忆管理 | Vector DB + Redis | 长期/短期记忆 |
| 人工交互 | OpenClaw Channel | 检查点审核 |

### 3.4 上下文管理策略

**目标：上下文 < 16K tokens**

| 策略 | 实现方式 | 压缩率 |
|:---|:---|:---|
| 滑动窗口 | 保留最近N轮对话 | 50% |
| 关键信息提取 | 提取决策、结论、待办 | 70% |
| 分层记忆 | 短期→中期→长期 | 80% |
| 摘要生成 | 阶段性生成摘要 | 60% |

| 系统可用性 | > 99% | 系统正常运行时间 |
| 并发支持 | 10+ | 同时处理任务数 |
| **Agent Scope** | **main** | **Agent在主会话运行，非per channel** |

### 5.3 Agent Scope 配置

| 配置项 | 值 | 说明 |
|:---|:---|:---|
| scope | `main` | Agent在主会话中运行，全局共享 |
| channel绑定 | 无 | 不绑定特定channel，支持跨channel调用 |
| 会话隔离 | 否 | 所有channel共享同一组Agent实例 |

**Scope 说明**：
- `main`: Agent 在主会话中运行，所有 channel 共享同一组 Agent 实例
- `per channel`: 每个 channel 独立运行 Agent 实例（已弃用）
