# 动作迁移输入素材质量指南

动作迁移的输出质量 80% 取决于输入素材。本文档详细说明目标图片和参考视频的最佳实践与常见陷阱。

---

## 目标人物图片（image_url）

### 必须满足

| 要求 | 说明 | 不满足的后果 |
|------|------|-------------|
| **全身可见** | 至少到膝盖；舞蹈类必须含脚部 | 生成视频中下半身畸变、比例失调 |
| **清晰轮廓** | 人物边缘锐利，不与背景融合 | 姿态估计失败，产生边缘撕裂 |
| **手部可见** | 双手分开、未遮挡面部，手指清晰 | 手部伪影（多指、融合、消失） |
| **正面或微侧** | 0°-30° 偏转 | 极端侧面/背面无法建立准确的身体模型 |
| **分辨率** | 短边 ≥ 300px | 面部和细节模糊，身份一致性差 |

### 强烈建议

| 要求 | 说明 | 原因 |
|------|------|------|
| **紧身衣物** | 贴合身体的上衣、裤子 | 宽松衣物（长裙、大衣、斗篷）遮挡关节，导致动作畸变 |
| **简洁背景** | 纯色或低复杂度背景 | 复杂背景会被带入生成视频，产生伪影 |
| **均匀光照** | 柔和、无强烈阴影 | 硬阴影遮挡身体部位，影响姿态检测 |
| **自然站姿** | T-pose 或自然站立 | 与参考视频首帧姿态差异过大会导致起始帧跳变 |
| **高对比度** | 人物与背景色调分明 | 低对比度 → 轮廓提取困难 |

### 图片预处理建议

当用户图片不理想时，按以下顺序建议预处理：

1. **背景杂乱** → `meitu image-cutout --image {url} --json` 抠图，然后 `meitu image-edit --image {cutout_url} --prompt "place person on a clean white studio background" --json` 换背景
2. **分辨率低** → `meitu image-upscale --image {url} --json` 超分
3. **半身图** → 坦诚告知用户：半身图做全身舞蹈效果差，建议提供全身图；如果只做上半身动作（手势、表情）可以尝试
4. **多人合照** → 建议用户裁剪出单人区域

### 特殊类型图片

| 类型 | 可行性 | 注意事项 |
|------|--------|----------|
| 真人照片 | 最佳 | 清晰正面全身照效果最好 |
| 动漫/插画角色 | 良好 | 风格一致性好，但写实动作可能有违和感 |
| 3D 渲染角色 | 良好 | 关节清晰是优势 |
| Q版/大头角色 | 一般 | 比例非人体比例，动作迁移可能失真 |
| 宠物/动物 | 较差 | 骨骼结构不同，人体动作迁移效果不可预期 |
| 多人图片 | 不建议 | 会混淆目标，建议裁剪为单人 |

## 参考动作视频（video_url）

### 必须满足

| 要求 | 说明 | 不满足的后果 |
|------|------|-------------|
| **单人画面** | 只有一个人在执行动作 | 多人 → 动作混淆，无法确定跟踪目标 |
| **全身在画面内** | 从头到脚全程可见 | 出画 → 该段动作丢失或产生跳变 |
| **镜头稳定** | 固定机位或极缓慢平移 | 手持晃动 → 姿态估计引入噪声 |
| **人物可辨** | 光照足够，衣物与背景有对比 | 暗光/剪影 → 关键点检测失败 |

### 强烈建议

| 要求 | 说明 | 原因 |
|------|------|------|
| **时长 3-10 秒** | 最佳 sweet spot | <3s 动作不完整；>10s 时间一致性下降，伪影累积 |
| **中等速度** | 正常到稍慢的动作节奏 | 极快动作（breaking、杂技）→ 肢体重影和模糊 |
| **简洁背景** | 纯色或低复杂度 | 复杂背景干扰姿态检测准确度 |
| **无遮挡** | 人物肢体不被道具/家具遮挡 | 被遮挡的关节无法准确追踪 |
| **无特效** | 无滤镜、字幕、水印覆盖人物 | 视觉噪声干扰骨骼检测 |
| **无剪辑** | 连续镜头，无跳切 | 跳切导致帧间动作不连续 |

### 理想参考视频来源

| 来源 | 适用性 | 说明 |
|------|--------|------|
| 舞蹈教学视频（正面固定机位） | 最佳 | 干净背景、全身、稳定镜头 |
| TikTok/抖音舞蹈挑战 | 良好 | 通常全身、单人，但可能有水印 |
| 健身/瑜伽示范 | 良好 | 动作清晰，速度适中 |
| 手势/肢体语言演示 | 良好 | 适合半身动作迁移 |
| 综艺/直播片段 | 一般 | 镜头可能不稳定，多人入画 |
| 电影/MV 片段 | 较差 | 多机位剪辑、光影变化大 |

### 视频预处理建议

1. **过长** → 使用 ffmpeg 截取核心 5-8 秒：`ffmpeg -i input.mp4 -ss 00:00:02 -t 00:00:08 -c copy output.mp4`
2. **有水印** → 尽量选无水印版本；水印在角落且不覆盖人物可以接受
3. **多人** → 裁剪画面只保留目标人物区域
4. **不稳定** → 使用视频稳定工具处理后再用

## 图片-视频匹配原则

**体型匹配是最容易被忽视但影响最大的因素。**

| 原则 | 说明 |
|------|------|
| **体型比例** | 目标图片与视频中人物的身体比例应相近（高矮胖瘦）。差异过大会导致动作变形。 |
| **画幅一致** | 全身图配全身视频，半身图配半身视频。不要用全身舞蹈视频配半身照片。 |
| **起始姿态** | 目标图片的姿势尽量接近视频第一帧。差异越大，起始过渡越突兀。 |
| **风格一致** | 真人照片配真人视频效果最自然；动漫角色配真人视频也可行但风格会有差异。 |

## 常见质量问题与成因

| 问题 | 典型表现 | 主要成因 | 预防措施 |
|------|----------|----------|----------|
| 手指异常 | 多指、缺指、融合 | 手部姿态估计精度有限 | prompt 加 "realistic hands"；接受这是技术局限 |
| 面部漂移 | 面部在帧间变化 | 时间一致性不足 | 使用清晰正面照、均匀光照 |
| 肢体扭曲 | 不自然的弯折 | 极端姿态超出模型能力 | 避免杂技/breaking 等极端动作 |
| 边缘撕裂 | 人物边缘波动 | 输入图片轮廓不清晰 | 使用高对比度图片，必要时抠图换背景 |
| 闪烁跳变 | 亮度/纹理突变 | 快速动作或复杂背景 | 缩短视频、简化背景 |
| 衣物穿帮 | 衣物变形或消失 | 宽松衣物遮挡关节 | 选择紧身衣物的图片 |
| 比例失调 | 头大身小或反之 | 图片与视频人物比例差异大 | 匹配体型比例 |
| 动作丢失 | 部分时段角色静止 | 人物在视频中出画 | 确保全程在画面内 |
